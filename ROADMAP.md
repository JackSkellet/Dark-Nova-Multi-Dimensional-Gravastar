# Roadmap

## Phase 0: Bootstrap

- Create the repository structure and reproducible Python environment.
- Record the live environment without hostnames, usernames, secrets, or private identifiers.
- Add fast tests and a CPU smoke experiment command.

## Phase 1: Definitions

- Formalize H1-H5 and their null hypotheses.
- Define quality, storage, runtime memory, active parameters, latency, continual adaptation, carrier pathways, semantic routes, and security failure.

## Phase 2: Prior Art

- Maintain `research/prior_art.csv`, `research/references.bib`, and the narrative review in `research/03_prior_art.md`.
- Pin DS4 source analysis to a specific commit.

## Phase 3: Component Evidence

- Expand E1-E5 from synthetic checks into public-repository and small-model experiments.
- Add stronger baselines before evaluating combinations.

## Phase 4: Integration Gate

- Build an integrated prototype only if component experiments show a reason to combine components.

## Phase 5: Assessment

- Decide go, revise, or stop for each research direction based on matched baselines and security constraints.
